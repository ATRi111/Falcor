# 2026-04-06 Notebook Weekly Outline Reset

- 生成毕设周记时，不能机械按最近几天的提交密度压缩成“高速推进”叙事；用户给出的阶段大纲比提交节奏更优先，周次要按“调研、实现、验证、优化”自然展开。
- 当用户只校正周记主线时，应先删除上一版周记和日记样稿，再仅重建周记，等周次节奏确认后再扩写日记，避免两套内容同时返工。
- 周记尾段如果对应毕业设计收尾，必须按“课题目标已按预期完成”来写，而不是继续停留在未完成优化；同时叙事应尽量向开题报告中的“近景Mesh、远景体素、自动LoD切换”主线靠拢。
- 混合渲染从整景Blend转向对象级路由的工作量不能压缩成“一周内发现问题并全部重构完成”；更合理的周记节奏是前一周先暴露性能瓶颈，后两周分别承担方案转向、实现细化和联调收尾。
- 周记语气不要写得像总结报告，尽量用学生自己记录进度的口吻去写，句子可以直接一点，但不要太随意，也不要堆“语义、收敛、闭环”这种明显偏AI的词。
