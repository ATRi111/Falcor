Voxel direct-light + AO render flow plan for AI

Goal
- Add a separate voxel rendering flow that uses existing voxel cache files.
- No GI / multi-bounce path reuse at runtime; render only direct lighting + AO in the voxel world.
- Keep changes small and localized.
- Reuse existing Falcor render-pass framework, Voxelization plugin, ReadVoxelPass, Python render-graph scripts, and AccumulatePass.

Design constraints
- Do not change the voxel file format.
- Do not rewrite voxelization or the cache reader.
- Do not break the current GI path in RayMarchingPass unless a tiny shared-helper refactor is clearly safe.
- Prefer adding a new pass over heavily editing RayMarchingPass behavior.

Recommended architecture
1. Keep cache generation and cache reading as-is
- Reuse VoxelizationPass_GPU and VoxelizationGPU.exe unchanged for writing caches.
- Reuse ReadVoxelPass unchanged for loading:
  vBuffer
  gBuffer
  pBuffer
  blockMap

2. Add a new pass dedicated to simple shading
- Recommended name:
  VoxelDirectAOPass
- Put it into the existing Voxelization plugin/library, not a new plugin.
- Its inputs should match RayMarchingPass:
  vBuffer
  gBuffer
  pBuffer
  blockMap
- Its output should be:
  color

3. Add a new Python render graph
- Recommended script:
  scripts\Voxelization_DirectAO.py
- Base it on scripts\Voxelization_GPU.py, but replace RayMarchingPass with VoxelDirectAOPass.
- Keep RenderToViewportPass, AccumulatePass, and ToneMapper.

Why a new pass instead of editing RayMarchingPass
- The user wants another rendering flow, not a rewrite of the current GI path.
- A new pass avoids destabilizing existing GI/debug behavior.
- The graph change becomes one-pass-for-one-pass.
- ReadVoxelPass and the cache contract stay untouched.

What to reuse directly
- Render-pass skeleton and fullscreen execution pattern:
  Source\RenderPasses\Voxelization\RayMarchingPass.cpp
  Source\RenderPasses\RasterPass\MyRasterPass.cpp
- Scene/light sampling helpers:
  Source\RenderPasses\Voxelization\MixLightSampler.slang
- Voxel BSDF / coverage / internal visibility:
  Source\RenderPasses\Voxelization\Shading.slang
- Voxel structs and grid helpers:
  Source\RenderPasses\Voxelization\VoxelizationShared.slang
  Source\RenderPasses\Voxelization\Voxel\VoxelGrid.slang
- Cache loading:
  Source\RenderPasses\Voxelization\ReadVoxelPass.cpp
- Existing graph wiring style:
  scripts\Voxelization_GPU.py

Implementation strategy
Phase 1: Pass skeleton
- Add:
  Source\RenderPasses\Voxelization\VoxelDirectAOPass.h
  Source\RenderPasses\Voxelization\VoxelDirectAOPass.cpp
  Source\RenderPasses\Voxelization\VoxelDirectAO.ps.slang
- Register the new pass in:
  Source\RenderPasses\Voxelization\VoxelizationBase.cpp
- Add files to:
  Source\RenderPasses\Voxelization\CMakeLists.txt
- First milestone: compile and show a flat debug color after first-hit traversal.

Phase 2: Reuse traversal / hit logic
- Reuse the current voxel traversal model from RayMarching.ps.slang:
  isSolidVoxel()
  isSolidBlock()
  checkPrimitive()
  rayMarching_*()
  calcLightVisibility()
- Preferred approach:
  extract only traversal/visibility helpers into one shared include, then include it from both RayMarching.ps.slang and VoxelDirectAO.ps.slang.
- If that refactor proves risky, copy only the minimum traversal helpers into VoxelDirectAO.ps.slang and leave a clear TODO to unify later.
- Do not change the voxel cache layout for this.

Phase 3: Direct lighting only
- Reuse current light sampling helpers from MixLightSampler.slang.
- Reuse PrimitiveBSDF shading from Shading.slang.
- For the first usable version, compute:
  primary voxel hit
  one direct-light estimate
  shadow visibility via calcLightVisibility()
- Keep support for:
  point lights
  directional lights
  env map
  optional emissive lights
- Reuse the current stochastic sampling model and frame accumulation instead of inventing a new deterministic light integrator.

Phase 4: Add AO
- AO should be voxel-space AO, not mesh-accurate AO.
- Use the first-hit voxel normal from PrimitiveBSDF.surface.getMainNormal().
- Sample a hemisphere around that normal.
- For each AO sample, cast a short ray with rayMarching(from, dir, aoRadiusInVoxels, true).
- AO term can start as:
  visibleSampleCount / totalSampleCount
- Then modulate direct lighting by AO, or expose AO-only and combined modes.
- Keep AO parameters small and explicit:
  aoEnabled
  aoSampleCount
  aoRadius
  aoStrength

Phase 5: Python graph
- Add scripts\Voxelization_DirectAO.py.
- Start from scripts\Voxelization_GPU.py.
- Replace only:
  RayMarchingPass -> VoxelDirectAOPass
- Keep:
  ReadVoxelPass
  RenderToViewportPass
  ToneMapper
- Keep AccumulatePass and set:
  enabled = True
  autoReset = True
  precisionMode = Single
- Reason:
  direct light from env/emissive sampling and stochastic AO will otherwise be noisy.

Recommended initial graph
- Keep VoxelizationPass_GPU in the script only if needed for the existing dummy edge into ReadVoxelPass.
- Treat it as a compatibility helper, not part of the new shading path.
- The real shading path should be:
  ReadVoxelPass -> VoxelDirectAOPass -> RenderToViewportPass -> AccumulatePass -> ToneMapper

Suggested pass properties / UI
- shadowBias
- useMipmap
- checkEllipsoid
- checkCoverage
- checkVisibility
- useEmissiveLight
- aoEnabled
- aoSampleCount
- aoRadius
- aoStrength
- drawMode with at least:
  Combined
  DirectOnly
  AOOnly
  NormalDebug
  CoverageDebug

What not to do
- Do not modify VoxelizationPass_GPU just for the new renderer.
- Do not change ReadVoxelPass binary parsing unless a bug is proven.
- Do not add a new voxel file format for “direct/AO only”.
- Do not rebuild a separate scene-loading or app framework when Mogwai + Python graph already solve the orchestration.

Validation checklist
1. Build only the affected targets first:
- Voxelization plugin
- Mogwai if needed

2. Functional smoke test
- Open Mogwai with the new script.
- Load an existing scene.
- In ReadVoxelPass, select an existing voxel cache and click Read.
- Verify the new pass renders without touching GI code paths.

3. Visual checks
- DirectOnly mode shows lit voxels and shadows.
- AOOnly mode darkens cavities and nearby occluded regions.
- Combined mode = direct * AO or direct + AO term, according to the chosen simple model.
- Turning off AO changes only the AO contribution.
- Existing RayMarchingPass graph still works.

4. Stability checks
- Changing the selected voxel file should recompile correctly through ReadVoxelPass.
- AccumulatePass should reset on camera/pass-option changes when autoReset is enabled.
